Level 2 Header

  • Introduction
    Level 2 header is a standard header now. LHA became popular then LHA was ported to other OS than MS-DOS. Restriction of filename length in Level 0 Header became a problem. Another problem is that Level 0 Header has only the infomation which depends on MS-DOS. Level 2 Header was defined in order to solve these problems. Severe restriction of filename length like Level 0 header go away by storing filename in extended header. Moreover, Level 2 Header can contain a new information without changing the format of the base header by not including the OS dependence information in base header.

  • Construction
    offset caption description
    00 Size of header(L) This is the byte count of whole header (which is sum of the byte count of base header and the total of byte count of each extended headers). Size of header is 0x002E(46) in the foregoing sample.
    01 Size of header(H)
    02 Compress method(1) Compress method identifies the compression algorithm of compressed data which is following this header. The format of compress method is "-???-". (the 1st byte and the 5th byte must be '-', and the length must be 5bytes) Compress method is "-lh0-" which means no compression, in the foregoing sample.
    03 Compress method(2)
    04 Compress method(3)
    05 Compress method(4)
    06 Compress method(5)
    07 Compressed size(LL) This is the byte count of compressed file which is following this header. Compressed size is 0x00000000 in the foregoing sample.
    08 Compressed size(LH)
    09 Compressed size(HL)
    10 Compressed size(HH)
    11 Original size(LL) This is the byte count of the file before the compression. Original size is 0x00000000 in the foregoing sample.
    12 Original size(LH)
    13 Original size(HL)
    14 Original size(HH)
    15 Last modified date and time(LL) This is the count of the elapsed seconds from 0:0:0 on january 1 1970 UTC to the last modified time of file. When the time zone is not correctly set up, you can interconvert with an uncertainty of maximum of 24 hours local time and UTC. Last modified date and time is 16:30:20 on Jul 7, 2002 in the foregoing sample.
    16 Last modified date and time(LH)
    17 Last modified date and time(HL)
    18 Last modified date and time(HH)
    19 Resurved You must set 0x20 which means normal file in MS-DOS attribute.
    20 Header level You must set 0x02 which means header level.
    21 CRC16(L) This is CRC16 of the file before the compression. CRC16 is 0x0000 in the foregoing sample.
    22 CRC16(H)
    23 OS ID This is a identifier of Operating System which compressed the file. OS ID is 'J'(0x4A) which means Java in the foregoing sample.
    24 Size of next extended header(L) This is the byte count of following extended header. Set this field as 0, when there is not following extended header. Size of next extended header is 0x0005 in the foregoing sample.
    25 Size of next extended header(H)

  • About Size of header
    In LHA, the head of the base header must not be 0x00 (since it means the end of archive). Therefore, when low byte of Size of header is 0x00, you must add a extra byte(it seems that 0x00 is added, usually) to tail of header and set low byte of Size of header as 0x01.
